## Behavior when a permission is missing

When a Graph / EXO / Purview API call fails due to a missing permission, the responsible collector emits `Status='Unknown'` per `docs/CHECK-STATUS-MODEL.md`. The `CurrentValue` field surfaces what could not be collected, and `Remediation` calls out the exact missing permission so the user knows what to grant.

Concretely:

| Failure mode | Status emitted | Counted toward Pass% denominator? |
|---|---|---|
| Missing Graph scope (delegated) | Unknown | No |
| Missing Graph app role (app-only) | Unknown | No |
| EXO cmdlet not available (license/role) | NotLicensed | No |
| Tenant does not run the relevant service | NotApplicable | No |
| Section explicitly skipped via `-Section` filter | Skipped | No |
| Transient API error (timeout, throttle) | Unknown | No |

See `docs/CHECK-STATUS-MODEL.md` for the full status taxonomy and decision tree.
